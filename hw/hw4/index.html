Homework Four

Developing an OWL Ontology in Protege

out Wednesday 2019-11-06, due Monday 2018-11-18

This homework will give you some experience in using the Protege editor to develop a simple OWL ontology with some example data.

Getting started

Here are some preliminary steps to get you ready to do the actual assignment, which is relatively simple.

  • Explore the Protege web site
  • Download and install the latest release. If you want to use the OWLViz panel which will generate diagrams of parts of ontologies, you will probably have to install the graphviz package. See the OWLViz information. You will need to also install one of the reasoning plugins. We recommend Pellet or Hermit.
  • Read Getting Started with Protege Desktop Editor which is designed to help get you up and running quickly
  • Look at Ontology Development 101: A Guide to Creating Your First Ontology which general advice and suggestions on ontology development

The Ontology

Build a simple ontology for the following conceptual model of the world. Parts may be underspecified. If you think this is the case, use common sense to fill in missing aspects. There are many ways to express most of these constraints that are logically equivalent. Choose the way that seems succinct and intuitive. You can, and in many cases, should use different names for your classes and properties to make their meaning be more intuitive to an English speaker. For example, you might use hasMother for the property linking a person to her mother rather than mother. Some guidance on best practices for vocabulary names is here.

The following statements describe the ontology you should create. It's intentionally given in an unstructured, redundant and rambling form, because this is often how a "customer" or domain expert might describe a set of concepts and relations. So you first step is probably to assemble and organize list of the classes and properties that are needed. Then you can give them names that are appropriate for an OWL ontology. Then you can identify the detailed axioms/constratins that each should have. Finally, you can enter them into Protege. As you do this, you will probably want to add some data to test your ontology and discover issues.

    All things in the world are either physical objects or abstract objects. Both kinds of things can have any number of name properties, whose values are strings. Nothing can be both a physical object and also an an abstract object.

    Physical objects can have a location property, but abstract objects cannot. A location's value must be a kind of place, which is a type of abstract object whose properties include a longitude and a latitude, both of which are real numbers.

    All physical objects are either an animate object or an inanimate object. Nothing can be both. Some animate objects are sexual-reproducers and some are not. All sexual reproducers are animate objects. Every sexual-reproducer is either a male or a female and none is both and only sexual-reproducers can be male or female.

    A male thing has a sex property whose value is the string "male" and a female thing has sex property whose value is "female". Moreover, if a thing has a sex property the value must be one of the strings "male" or "female". All things that have a sex property with the value "male" are male things. All things that have a sex property with the value "female" are female things.

    One type of animate object is a person. A person can have many properties. Every person has exactly one age property, whose value is an integer, exactly one mother property, whose value must be a kind of woman, exactly one father, whose value must be a kind of man, exactly one social security number, whose value is a string. No two people can have the same social security number.

    A person can have any number of employer properties whose values must be a kind of organization. An organization is a kind of abstract object. Infants cannot have an employer.

    A person has exactly two parent properties, one if which is the person's mother and the other the person's father. The parent of a parent of a person is the person's grandparent. A father of a parent is a grandfather and the mother of a parent is a grandmother.

    A person is a sexual-reproducer. A man is a person who is also a male. A woman is a person who is also a female. We define an infant as any person with an age whose value is between 0 and 1 (inclusive) and a child as a person whose age is between 2 and 18 (inclusive) and an adult as a person older than 18. Every person is either an infant, a child or an adult. No person is more than one of these subtypes.

    A boy is defined as a person who is also a male thing and not an adult. A girl is a female thing and a person who is either an infant or a child.

What and how to submit

Click on TBD to accept the assignment and have github create a repository that you can put your results into. You can checkout the peeps ontology at https://github.com/UMBC-CMSC-491-691-F18-Knowledge-Graphs/peeps.git which includes an ontology (peeps.ttl) and a file of instances (mypeeps.ttl). These can be used as examples to get you started. We recommend that you load peeps.ttl and mypeeps.ttl into Protege and view the data and schema, rather than just looking at the serialization (which is in Turtle).

You must run the Pellet reasoner to ensure that there are no contradictions and all appropriate inferences are drawn.

The HW4 repository will come with stubs for three files you will complete for the homework:

  • myontology.ttl - the ontology
  • mydata.ttl - a file importing the ontology and including data on instances. You should include at least ten people and at least two instances of each of the types organizations, places, males, females, infants, children and adults. Your data should have examples of all of the appropriate properties.
  • documentation.md -- a file in markdown notation describing your ontology and any choices you made that were interesting.
